能力边界

项目定位保持清晰，避免把桌面管理工具扩成另一套中转站服务端。

  • 只做桌面 App BalanceHub 基于 Tauri 运行在本机，不提供 Web 自部署版本。
  • 当前只展示 NewAPI AnyRouter 按 NewAPI 方言兼容处理；sub2api 尚未接入。
  • 凭据本地保存 Cookie、Token、API Key 和中转站配置保存在当前电脑，不交给远端服务。
  • 只接受 Issues 仓库使用 Issues 收集问题和建议，不接受 Pull Request。

自动更新

`.sig` 文件用于应用内自动更新签名校验，不是手动安装包。应用会读取 GitHub Releases 的 `latest.json`，下载当前平台匹配的更新包，校验后安装并重启。

常见问题

几个下载和使用时最容易混淆的点。

Release 里的 `.sig` 要下载吗？

不用。它是应用内自动更新用的签名文件，不是安装入口。

为什么系统提示未知开发者？

当前安装包通过 GitHub 分发，但没有做系统级付费代码签名。macOS 或 Windows 可能会显示系统信任提示。

API Key 能不能代表账号余额？

不能。API Key 查询到的是 Key 维度额度，不等同于账号总额度，也可能返回无限额度。
