# 中转站配置

BalanceHub 当前只支持 NewAPI 兼容中转站。AnyRouter 这类站点按 NewAPI 接口方言处理，不在界面上作为独立中转站类型展示。

中转站配置和凭据只保存在当前电脑的应用数据目录中。导出配置会包含中转站信息和凭据，请只在可信设备之间迁移。

## 认证方式

认证优先级建议是：账号密码 > Cookie > 访问令牌 > API Key。账号密码可以登录站点，并按需补全后续会话、访问令牌和 API Key。

### 账号密码

账号密码适合首次配置或希望由应用建立会话的场景。应用只在登录站点时使用账号密码，登录成功后缓存 Session，并可继续补全访问令牌和 API Key。

### Cookie

Cookie 适合完整账号能力，包括读取账号信息、额度、签到、API Key 管理、请求日志等。

使用 Cookie 时，用户 ID 可以由应用自动解析，通常不需要手动填写。

### 访问令牌

访问令牌适合不想长期使用 Cookie 的场景。使用访问令牌时，用户 ID 需要明确填写或由同步流程补齐。

如果当前中转站支持令牌创建，但本地配置里缺少令牌，应用会在需要同步时提示是否创建，不会静默创建。

### API Key

API Key 只适合 Key 维度的额度查询和相关操作，不等同于账号登录态。

使用账号凭据补全配置时，应用会先读取并保存站点已有的 API Key（最多 100 个）。存在多个 Key 时会保留完整列表，并要求选择一个主 Key；只有确认列表为空时才提示创建。之后在密钥管理中刷新、创建或删除 Key，也会用最新查询结果同步本地缓存。

使用 API Key 时要注意：

- 不支持签到。
- 不代表账号总额度。
- 某些中转站返回的 Key 额度上限可能是无限。

## 连接测试

测试连接会尽量完成这些检查：

- 中转站地址是否可访问。
- 站点信息是否可以读取。
- 当前认证方式是否有效。
- 账号或 API Key 额度接口是否可用。

如果测试连接拿到了额度结果，应用会缓存这次结果，保存后卡片可以直接展示最新额度。
