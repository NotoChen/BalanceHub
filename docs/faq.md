# 常见问题

## `.sig` 文件是什么？

`.sig` 是应用内自动更新使用的签名文件，不是安装包。手动下载安装时不需要打开它。

## 应用内自动更新会怎么执行？

应用会从 GitHub Releases 获取最新版本信息，下载当前平台匹配的更新包，校验 `.sig` 签名后安装，并在完成后重启应用。

## 为什么 macOS 或 Windows 提示未知开发者？

当前安装包通过 GitHub Releases 分发，但没有做系统级付费代码签名。macOS 和 Windows 可能会提示未知开发者或 SmartScreen 拦截。

这是系统信任提示，不代表安装包一定损坏。后续如果项目用户规模稳定，再考虑付费代码签名。

## 为什么签到记录为空？

部分中转站不提供官方签到记录接口。BalanceHub 会优先读取官方记录；如果远端不支持，则使用本地记录兜底。本地记录只能覆盖应用内执行过的签到。

## 为什么 API Key 查到的是无限额度？

部分中转站的 API Key 额度上限可能没有限制，接口会返回无限或无法计算的上限。BalanceHub 会按无限额度展示。
