# 更新记录

BalanceHub 的重要变更会记录在这里。

## 0.3.2

### 新增

- 中转站卡片补充 NewAPI 用户名，并重新组织账号、余额、认证方式和运行状态的信息层级。
- 只读解析 Codex 与 Claude Code 配置文件，根据中转站地址和 API Key 识别当前默认中转站。
- 新增临时 CLI 实例管理：记录 CLI 类型、终端、工作目录、PID、启动与退出时间、退出码，并支持重新启动。
- macOS 使用 Ghostty 启动临时 CLI 时记录稳定终端标识，可从实例抽屉精确打开对应窗口。

### 变更

- 中转站卡片改为可扩展的状态标签，不再平铺展示全部能力。
- 测活只保留 Codex 与 Claude Code CLI，移除 HTTP 测活调用和相关展示。

## 0.3.1

### 修复

- 修复 NVM 安装的 Codex CLI 被解析为包内脚本后无法找到 Node，导致本地扫描误判为不可用的问题。

## 0.3.0

### 新增

- 设置中新增「自启后最小化」开关：开机自启动时不弹出主窗口，仅保留托盘图标，点击托盘随时打开。

### 变更

- macOS：关闭主窗口后自动隐藏 Dock 图标，应用退为纯托盘形态；从托盘重新打开时恢复显示。

## 0.2.0

### 修复

- 自动签到失败不再全天反复重试并连发通知：改为每天最多 3 次、间隔 30 分钟，当天仅首次失败发一条通知。
- 后台自动刷新不再覆盖同一时间用户正在编辑保存的中转站配置；设置抽屉打开期间，后台同步不再清空未保存的修改。
- 授权自动测活后再保存设置，不再丢失授权记录。
- 测活 CLI 超时改为按进程组整树终止，不再遗留继续消耗额度的孤儿进程。
- 测活运行期间不再因 CLI 自动探测卡住其他操作。
- 修复 AnyRouter 子路径部署时接口地址被截断的问题。
- 手动刷新失败现在会明确提示原因；通知测试逐个渠道报告失败，不再被“部分成功”掩盖。
- 看板已有数据时，后台一次瞬时加载失败不再切换到全屏错误页。

### 新增

- 配置结构升级迁移：旧版本 data.json 会先自动备份再迁移，不再要求重新初始化。
- 站点刷新失败、测活由正常转为失败时发送通知（状态翻转只发一次，不重复打扰）。
- 设置中新增「重置自动测活授权」入口；开关已开但授权缺失时提供重新授权入口。
- HTTP 测活结果区分凭据失效与限流/站点故障，避免瞬时故障被误读为账号失效。
- 启动时自动清理残留的临时 CLI 目录，临时凭据文件权限收紧为仅当前用户可读。

### 变更

- 手动刷新与自动调度刷新互斥，避免对同一批站点重复请求；批量刷新与测活改为滚动并发。

## 0.1.0

### 新增

- NewAPI 兼容中转站账号管理。
- 余额刷新、签到、用量趋势、请求日志、API Key 管理和修改密码流程。
- Codex / Claude Code 测活，支持中转站单独覆盖全局配置。
- 配置导入导出，支持跨设备迁移。
- 系统通知和 Webhook 通知。
- 基于 GitHub Releases 和 Tauri Updater 的自动更新。
- macOS、Windows、Linux 的 x64 和 ARM64 发布包。
